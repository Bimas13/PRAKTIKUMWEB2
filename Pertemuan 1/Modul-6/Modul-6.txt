Saat pengguna membuka situs.com/mahasiswa/index, browser mengirim request yang ditangkap oleh Front Controller / Router (mis. public/index.php + App.php) yang memecah URL untuk menentukan Controller Mahasiswa dan method index(). Controller memanggil Model untuk mengambil data mahasiswa dari database (CRUD), menerima data bersih dari Model, lalu memilih View yang sesuai dan mengirimkan data tersebut ke View. View hanya bertugas merender HTML/CSS dan mengembalikan respons ke browser — sehingga alurnya ringkas: request → router/front controller → controller → model → controller → view → response.
